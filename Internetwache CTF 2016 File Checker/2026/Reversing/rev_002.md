## 📌 개요

이 문제는 ELF 64비트 바이너리를 리버싱하여 플래그를 찾는 간단한 리버싱 문제입니다.  
바이너리는 존재하지 않는 `.password` 파일을 읽으려 시도하며, 정적 분석을 통해 플래그 관련 문자열을 추출할 수 있습니다.  
`strings`, `ltrace`, `radare2` 등의 도구를 활용한 정적 및 동적 분석이 핵심입니다.

## 🧠 문제 분석

바이너리는 `.password` 파일을 열어 각 바이트를 순차적으로 비교합니다.  
비교 대상 바이트는 바이너리 내부에 하드코딩되어 있으나, 직접 `strings`로는 확인되지 않습니다.  
대신, `radare2`나 `hexdump`를 통해 해당 바이트 시퀀스를 추출하거나,  
실행 시 비교 로직을 분석해 원본 플래그를 복원해야 합니다.

이전 `strings` 출력에서 `.password`와 `Congr`이 함께 나타난 점, 그리고 `ltrace` 결과를 종합하면,  
`.password` 파일에 올바른 비밀번호를 넣으면 `Congratulations!` 메시지와 함께 플래그가 출력될 가능성이 큽니다.

또한, `radare2` 분석에서 비교되는 바이트 시퀀스는 다음과 같을 것으로 추정됩니다:

```
\x12\xee\x12\xe0\x12\xbc\x12\xf1\x12\xee\x12\xeb\x12\xf2\x12\xd8\x12\xf4\x12\xef\x12\xd2\x12\xf4\x12\xec\x12\xd6\x12\xba
```

이 값들은 단순한 인코딩(예: XOR, 덧셈 기반)을 사용했을 가능성이 높습니다.  
각 바이트에 인덱스나 누적 값을 더해 ASCII 범위(특히 0x20~0x7e)로 변환하는 방식일 수 있습니다.

## 💥 익스플로잇

먼저 인코딩된 데이터를 추출하고, 복호화 스크립트를 작성합니다.

```bash
cd /tmp/ctf && printf "\x12\xee\x12\xe0\x12\xbc\x12\xf1\x12\xee\x12\xeb\x12\xf2\x12\xd8\x12\xf4\x12\xef\x12\xd2\x12\xf4\x12\xec\x12\xd6\x12\xba" > password_data.bin && xxd password_data.bin
```

**출력:**
```
00000000: 12ee 12e0 12bc 12f1 12ee 12eb 12f2 12d8  ................
00000010: 12f4 12ef 12d2 12f4 12ec 12d6 12ba       ................
```

각 값은 2바이트로, 리틀엔디안 형식으로 해석해야 할 수 있습니다.  
하지만 실제로는 2바이트 값 그대로 사용됩니다
