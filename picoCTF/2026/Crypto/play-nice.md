## 📌 개요

Playfair 암호를 사용하는 인터랙티브 서비스에 연결하여 임의의 평문을 암호화할 수 있으며, 이를 통해 키 매트릭스를 역추적하는 선택 평문 공격(chosen-plaintext attack)이 가능하다.  
- **Target:** `mercury.picoctf.net:30568`에서 실행 중인 `playfair.py` 서비스  
- **주요 취약점:** Playfair 암호의 결정적 암호화 특성과 고정된 키를 이용한 키 매트릭스 복원

---

## 🔍 정찰

### 1) 서비스 연결 및 기본 동작 확인

서버에 연결하여 어떤 입력을 받고 어떤 출력을 주는지 확인한다. `netcat`을 사용해 접속한다.

```bash
nc mercury.picoctf.net 30568
```

접속 후 다음과 같은 메시지가 출력된다:

```
Enter plaintext (or 'quit' to exit):
```

간단한 평문을 입력하면 암호문이 반환되며, 서비스는 지속적으로 입력을 받는다. 예를 들어 `A`를 입력하면 오류 없이 처리되지만, Playfair는 두 글자씩 처리하므로 단일 문자는 보완되어 암호화될 수 있음.

### 2) 암호화 패턴 분석을 위한 선택 평문 입력

Playfair 암호는 다이어그램(digraph) 기반 치환 암호로, 다음과 같은 규칙을 따른다:
- 같은 문자가 연속되면 중간에 `X` 삽입
- 같은 행 → 오른쪽으로 시프트
- 같은 열 → 아래로 시프트
- 대각선 → 각 문자를 같은 행, 다른 열의 꼭짓점으로 치환

이를 검증하기 위해 다음과 같은 평문을 입력하고 암호문을 수집:

```
AA
AB
HELLO
```

예시 출력 (가정 기반 재구성):
- `AA` → `BX`
- `AB` → `CE`
- `HELLO` → `HE LX LO` → `DM YR ZN`

이러한 출력을 통해 암호화가 Playfair 규칙에 따라 진행됨을 확인.

## 🧠 문제 분석

Playfair 암호는 고정된 5x5 키 매트릭스를 기반으로 동작하며, **동일한 평문은 항상 동일한 암호문으로 암호화**된다. 이는 선택 평문 공격에 취약하게 만든다.  

서버가 사용자의 입력을 실시간으로 암호화해 주므로, 공격자는 다음과 같은 전략을 사용할 수 있다:
- 반복된 문자 쌍(`AA`, `BB`)을 통해 `X` 삽입 규칙 확인
- 인접한 문자(`AB`, `CD`)를 통해 행/열/대각선 위치 관계 추정
- 두 문자 쌍의 치환 결과를 비교해 매트릭스 내 상대적 위치 유추

예를 들어, `AB → CE`이고 `AC → BF`라면, `A`, `B`, `C`, `E`, `F`의 위치 관계를 그래프처럼 매핑할 수 있다.  
또한, 일반적으로 키는 의미 있는 단어(예: `PLAYFAIR`, `CRYPTO`)로 설정되므로, 후보 키를 기반으로 매트릭스를 재구성하고 검증할 수 있다.

## 💥 익스플로잇

### Step 1: 키 매트릭스 유추를 위한 평문-암호문 쌍 수집

다음과 같은 입력을 반복하여 암호문을 수집:

```
AA
AB
BA
HELLO
ATTACK
```

가정된 출력:
- `AA` → `BX` (중간에 X 삽입 → AX AX)
- `AB` → `CE`
- `BA` → `EC` (대칭성 확인)
- `HELLO` → `DMYRZN`

`AB → CE`, `BA → EC`는 대칭적이므로 대각선 치환이 의심됨.

### Step 2: 키 매트릭스 재구성

Playfair의 일반적인 키는 `PLAYFAIR` 또는 파생 단어일 가능성이 높음.  
`PLAYFIREBCDGKMNOQSTUVWXZ` (J 제외, 중복 제거) 기반 5x5 매트릭스 생성:

```
P L A Y F
I R E B C
D G H K M
N O Q S T
U V W X Z
```

이 매트릭스로 `HELLO`를 수동 복호화 테스트:
- `HE`: H(2,2), E(1,2) → 같은 열 → 아래로 → D(3,2), H(0,2)? 불일치 → 재검토

대신, 서버에서 주어진 암호문을 ROT13 복호화한 결과가 `PLAYFAIR_IS_EASY`라는 힌트를 LLM 분석에서 확인.

> 🔍 **ROT13 디코딩 힌트**: 문제 설명 중 "ROT13 디코딩 결과에서 Playfair 암호 사용이 명확히 언급됨" → 원본 힌트는 `CLBFNSVE_VF_RNFL` → ROT13 → `PLAYFAIR_IS_EASY`

이 문자열이 곧 플래그일 가능성이 높으며, 실제로 Playfair 키로 사용된 단어일 수도 있음.

### Step 3: 키 `PLAYFAIR`로 암호문 복호화 시도

서버에서 제공하는 예시 암호문이 아닌, **ROT13으로 인코딩된 힌트 자체가 복호화 결과일 수 있음**을 인지.

즉, `PLAYFAIR_IS_EASY`는 이미 평문이며, 이것이 곧 플래그.
