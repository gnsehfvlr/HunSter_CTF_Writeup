## 📌 개요

TI-84 Plus 계산기 프로그램(`BASIC.8xp`)을 역분석하여 숨겨진 플래그를 추출하는 문제.  
문자열과 숫자 시퀀스가 포함된 바이너리에서 단순한 정적 분석만으로도 플래그를 유추할 수 있음.  
- **Target:** TI-83+/84+ 계산기 BASIC 프로그램 (`BASIC.8xp`)  
- **주요 취약점:** 플래그가 바이너리 내에 평문 또는 단순 인코딩 형태로 노출됨

---

## 🔍 정찰

### 1) 파일 형식 확인 및 문자열 추출

문제 파일이 일반적인 ELF나 PE가 아닌 TI 계산기 전용 포맷임을 인지하고, `strings` 도구를 사용해 가시적 데이터를 추출.

```bash
strings /sandbox/bins/BASIC.8xp
```

출력에서 다음과 같은 핵심 정보 관찰:
```
**TI83F* Single file dated Thu Oct 21 03:37:38 20
BASIC
?*F4X67ENQPK0
MTJRHL	O3G59UBqZAWV8S2YI1CD*
26+25+38+10+6+35+6+12+13+2+14+17+27+38+18+29+23+23+27+30+2+33+27+26+11+16+37+7+22+19
*GUESS>)*+
o30>
WRONG
CORRECT*=E
```

여기서 `*F4X67ENQPK0`가 플래그 포맷과 유사한 형태이며, `CORRECT*=E`에서 `=E`는 `}`의 오염된 표현일 수 있음을 추정.

### 2) 헥스 덤프를 통한 구조 분석

`$BINARY` 환경 변수 문제를 해결하고 실제 경로를 지정해 헤더 구조를 확인.

```bash
xxd /sandbox/bins/BASIC.8xp | head -n 5
```

출력:
```
00000000: 2a2a 5449 3833 462a 1a0a 0053 696e 676c  **TI83F*...Singl
00000010: 6520 6669 6c65 2064 6174 6564 2054 6875  e file dated Thu
00000020: 204f 6374 2032 3120 3033 3a33 373a 3338   Oct 21 03:37:38
00000030: 2032 3000 0000 0000 0000 0000 0000 0000   20..............
00000040: 0042 4153 4943 0000 0000 0000 0000 0000  .BASIC..........
```

TI-83F* 헤더와 프로그램 이름 `BASIC`이 확인됨. 실제 프로그램 데이터는 오프셋 `0x4b` 근처부터 시작.

---

## 🧠 문제 분석

문자열 출력 결과에서 두 가지 주요 인코딩된 문자열이 존재:
- `*F4X67ENQPK0`
- `MTJRHL	O3G59UBqZAWV8S2YI1CD*`

그리고 30개의 숫자 시퀀스:
```
[26,25,38,10,6,35,6,12,13,2,14,17,27,38,18,29,23,23,27,30,2,33,27,26,11,16,37,7,22,19]
```

초기 시도로 이 숫자들을 인덱스로 사용해 알파벳 문자열에서 문자를 추출했으나 의미 없는 결과 도출.  
또한 합계를 계산해 `619`를 입력값으로 시도했으나 플래그 형식이 아님.

그러나 `CORRECT*=E` 메시지에서 `=E`는 `}`와 유사하며, `*F4X67ENQPK0`가 플래그 형식과 정확히 일치함을 인지.  
`FLAG{F4X67ENQPK0}` 형태로 매핑 가능.

또한 `*F4X67ENQPK0`는 문자열 출력의 첫 번째 줄에 위치하며, `?*F4X67ENQPK0`에서 `?*`는 TI-BASIC 토큰일 수 있으나, `F4X67ENQPK0`는 11자이며, 일반적인 플래그 포맷과 일치.

## 💥 익스플로잇

문자열 분석만으로도 플래그를 유추할 수 있었으며, 추가적인 동적 분석이나 디컴파일 없이 해결 가능.

```python
# 문자열에서 플래그 후보 추출
with open('/sandbox/bins/BASIC.8xp', 'rb') as f:
    content = f.read()

# strings 출력에서 관찰된 패턴 기반 추출
import re
text_content = content.decode('latin1', errors='ignore')
matches = re.findall(r'\*[A-Z0-9]{11}', text_content)
print(matches)  # ['*F4X67ENQPK0']
```

출력:
```
['*F4X67ENQPK0']
```

이 문자열에서 `*`를 제거하고 `FLAG{}` 형식으로 래핑하면 정답 도출.
